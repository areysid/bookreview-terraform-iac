# 🏗️ WHY TERRAFORM? WHY MODULES?

## ❓ Why Terraform?

Terraform lets you:

```text
Write infrastructure like code → reproducible, version-controlled, automated
```

Instead of manually clicking in AWS console:

* You define everything in `.tf` files
* Run `terraform apply`
* Entire infra spins up automatically

## ❓ Why modules?

Without modules:

```text
Everything in one file → messy, unscalable ❌
```

With modules:

```text
Split into reusable components ✅
```

## 🧱 Your modules

You created **5 core modules**:

---

### 1. 🌐 Network Module

Creates:

* VPC
* Subnets (web / app / db tiers)
* Internet Gateway
* NAT Gateway
* Route tables

👉 Output:

```hcl
vpc_id
subnet_ids_by_tier
```

---

### 2. 🔐 Security Group Module

Handles:

* Dynamic SG creation using `for_each`
* Ingress/egress rules
* SG-to-SG references

👉 Key concept:

```text
web SG allows traffic FROM ALB SG
app SG allows FROM internal ALB SG
db SG allows FROM app SG
```

---

### 3. 💻 EC2 Module

Creates:

* Web EC2 (public)
* App EC2 (private)

Uses:

```hcl
subnet_ids_by_tier["web"]
subnet_ids_by_tier["app"]
```

Also runs:

```text
user_data scripts (frontend + backend setup)
```

---

### 4. ⚖️ ALB Module

Creates:

* Public ALB → for frontend
* Internal ALB → for backend

Also:

* Target groups
* Listeners
* Attach EC2 instances dynamically

---

### 5. 🗄️ Database Module

Creates:

* RDS MySQL instance
* Subnet group (private subnets)

# 🔄 HOW MODULES CONNECT

```text
Network → provides subnets + VPC
        ↓
SG module → attaches to VPC
        ↓
EC2 → uses subnets + SGs
        ↓
ALB → uses subnets + SGs + EC2 instances
        ↓
RDS → uses subnets + SGs
```

👉 Everything is **loosely coupled but connected via outputs**

# ⚙️ KEY DESIGN DECISIONS

---

## 1. Tier-based subnet design

```hcl
subnets = {
  web = { public = true }
  app = { public = false }
  db  = { public = false }
}
```

👉 Enables:

* Public internet access only where needed
* Private isolation for backend + DB

---

## 2. `for_each` everywhere

Instead of:

```hcl
resource "aws_instance" "web"
resource "aws_instance" "app"
```

You used:

```hcl
for_each = var.ec2_instances
```

👉 Result:

* Scalable
* DRY (Don’t Repeat Yourself)
* Easily extendable

---

## 3. Dynamic wiring using maps

Example:

```hcl
subnet_ids_by_tier["web"]
```

👉 No hardcoding subnet IDs

---

## 4. Internal vs Public ALB

| ALB Type     | Purpose              |
| ------------ | -------------------- |
| Public ALB   | User traffic         |
| Internal ALB | Backend-only traffic |

👉 This is **real-world architecture**

# 🚧 MAJOR OBSTACLES & HOW YOU SOLVED THEM

---

## 🔴 1. Hardcoded values (early stage)

### Problem:

* Protocols
* Health checks
* Ports

### Fix:

* Parameterized via variables
* Made module reusable

---

## 🔴 2. Output refactoring

### Problem:

Old outputs:

```hcl
public_alb_dns_name
```

Didn’t work with `for_each`

### Fix:

```hcl
alb_dns_names = {
  web = ...
  app = ...
}
```

---

## 🔴 3. Subnet passing confusion

### Problem:

Where to define subnet_id?

### Solution:

```text
Networking module → outputs
Root → passes to EC2/ALB
```

👉 Correct separation of concerns

---

## 🔴 4. Security group dependency issues

### Problem:

SG referencing SG not yet created

### Fix:

```hcl
try(...)
depends_on
```

---

## 🔴 5. ALB target groups unhealthy

### Causes:

* App not running
* Wrong health check path
* SG misconfig

### Fix:

* Verified backend
* Adjusted health check to `/`

---

## 🔴 6. User data scripts not running properly

### Problem:

* Scripts not applied
* Instances not recreated

### Fix:

```text
terraform apply updates user_data
(or recreate instance)
```

---

## 🔴 7. RDS connection failing

### Problem:

```text
endpoint = host:3306 ❌
```

### Fix:

```hcl
split(":", endpoint)[0]
```

---

## 🔴 8. DB authentication failure

### Problem:

```text
password had special characters (#)
```

Shell interpreted it incorrectly

### Fix:

* Simplified password
  OR
* Used safer env handling

---

## 🔴 9. Internal ALB timeout

### Problem:

Frontend calling internal ALB directly

### Fix:

```text
Use Nginx proxy instead
```

---

## 🔴 10. `/api/api` duplication

### Problem:

```nginx
proxy_pass ... /api/
```

### Fix:

```nginx
proxy_pass ... (no /api)
```

---

## 🔴 11. Frontend hitting localhost

### Problem:

Fallback in code:

```js
|| "http://localhost:3001"
```

### Fix:

Removed fallback
